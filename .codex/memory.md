  lucide-vue-next icons, vue-sonner toasts.

- Current workspace note: this directory is not currently a Git repository.
- Current PHP environment has `PDO` and `pdo_mysql`, but not `pdo_sqlite`.
  Pest tests that use SQLite `:memory:` fail until SQLite support is enabled.

## Architecture

The app has four main areas:

- Public site: `/`, `/about`, `/contact`
- Admin panel: `/admin/dashboard`, `/admin/users`
- Auth pages: Fortify routes rendered through Inertia pages under
  `resources/js/pages/auth/`
- User settings: `/settings/profile`, `/settings/security`,
  `/settings/appearance`

Route entry points:

- `routes/web.php` defines public and admin routes.
- `routes/settings.php` defines authenticated settings routes.
- Fortify registers auth routes and renders custom Inertia views from
  `app/Providers/FortifyServiceProvider.php`.

Layout assignment is centralized in `resources/js/app.ts`:

- `Welcome` and `public/*` use `PublicLayout`.
- `auth/*` uses `AuthLayout`.
- `settings/*` uses `[AdminLayout, SettingsLayout]`.
- Everything else uses `AdminLayout`.

## Backend Notes

- `app/Models/User.php` uses attribute-based fillable/hidden definitions and
  includes Fortify two-factor auth.
- `app/Providers/AppServiceProvider.php` sets Carbon immutable dates, blocks
  destructive DB commands in production, and tightens password rules in
  production.
- Admin controllers:
  - `DashboardController` returns user stats.
  - `UserController` returns latest users paginated by 15.
- Public controller:
  - `ContactController` validates contact form input and returns an Inertia
    flash toast. Actual mail sending is still TODO.
- Public user registration is disabled in `config/fortify.php`. The initial
  admin user is seeded by `DatabaseSeeder`.

## Frontend Notes

- Main alias: `@/` maps to `resources/js/`.
- Shared Inertia props come from `HandleInertiaRequests`:
  - `name`
  - `auth.user`
  - `sidebarOpen`
- Admin chrome components live in `resources/js/components/admin/`
  (sidebar, topbar/navbar, footer nav, breadcrumbs, and user menu).
- Admin sidebar navigation lives in
  `resources/js/components/admin/AppSidebar.vue`.
- UI primitives live in `resources/js/components/ui/`.
- Use `cn` from `resources/js/lib/utils.ts` for conditional Tailwind classes.
- Flash toasts use this backend shape:

```php
return back()->with([
    'toast' => ['type' => 'success', 'message' => 'Message text'],
]);
```

Valid toast types: `success`, `info`, `warning`, `error`.

## Public Landing

- `/` is a standalone company landing for Solucion Digital in
  `resources/js/pages/public/Welcome.vue`.
- The landing includes its own header, footer, portfolio filter, and contact
  form. `PublicLayout.vue` skips its wrapper on `/` to avoid duplicate chrome.
- The landing contact form posts to `/contact` through Inertia and sends
  `name`, `email`, `phone`, `business_type`, and `message`.
- The landing now positions Solucion Digital as a software development company
  with SaaS system rental, distributor SaaS plans, chatbot/automation, and
  business systems for restaurants, retail, pharmacy, hardware, and veterinary
  workflows.

- `resources/js/wayfinder/`

They are regenerated by the Wayfinder/Vite tooling.

## Adding Admin Sections

When adding a new admin feature, follow this local pattern:

1. Add routes under the authenticated `admin.` route group in `routes/web.php`.
2. Add the controller in `app/Http/Controllers/Admin/`.
3. Add the page in `resources/js/pages/admin/`.
4. Add sidebar navigation in `resources/js/components/admin/AppSidebar.vue`.
5. Add focused Pest coverage when behavior or access control changes.
6. Run relevant checks, usually `composer test` and `npm run types:check`.

## Working Preferences

- Document all implemented features, decisions, and learning notes inside

- Update this memory whenever we establish a new convention, add a major
